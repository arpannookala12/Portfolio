Projects

Integrating IoT, ML, and Advanced Analytics

Module 1: Early Explorations & IoT

1. Pot-in-Pot Refrigerator (Jan – Mar 2020)

Short Description: Built an eco-friendly refrigerator using two clay pots and integrated an IoT module (Bolt IoT) to log hourly temperatures, then applied polynomial regression to predict future temperatures.

Objective: Create an environmentally sound fridge by placing a smaller clay pot inside a larger one, sealing the gap with sand and watering it occasionally to keep temperatures low. Leveraged Bolt IoT to collect temperature data and employed polynomial regression to predict temperature variations.

  • Tools & Tech: Bolt IoT, Polynomial Regression, Basic Analytics
  • Outcome: Learned foundational data collection & ML concepts, bridging physical engineering with data-driven insights.

2. Fire Detection & Localization (Jan – Jun 2021)

Short Description: A rapid fire detection system leveraging Google’s Inception V3 + IoT. Achieved ~87% validation accuracy on Fire/Smoke/Neutral scenes, integrating Telegram alerts for real-time notifications.

Objective & Approach: Developed a deep learning model (Inception V3) trained on images of fire, smoke, and neutral scenes to accurately detect fire in indoor/outdoor environments. Combined with a Bolt Wi-Fi module and a piezo buzzer for hardware alerts, plus real-time video analysis (OpenCV).

Dataset & Augmentation: 3,000 images (Fire, Smoke, Neutral) with flipping and zoom-based augmentations. Achieved 91.05% training accuracy, 87.23% validation after 10 epochs.

System Integration: Sends alert messages + captured frames to a Telegram chatbot when detection confidence exceeds threshold. Overfitting mitigated via dropout layers; Inception V3’s sparsely connected architecture optimized computational efficiency.

Future Scope: Scalability with multiple IoT devices, improved localization with bounding boxes and GPS data.

Module 2: Jumping into ML & AI

3. Credit Card Fraud Detection (Oct – Dec 2023)

Short Description: Logistic regression with SMOTE to tackle class imbalance, achieving 95% ROC AUC. Operationalized model to categorize transactions by risk level.

Data & Methods: Worked on a dataset of credit card transactions from European cardholders. Engineered features (binning, WoE, IV) to improve model performance. Achieved a 95% ROC AUC by addressing significant class imbalance via SMOTE.

Deployment & Visualization: Provided donut charts for interpretability, dividing outcomes into risk categories for real-world usage. Managed version control with Git and continuously refined features for better fraud detection.

4. Formula 1 Data Wrangling & Analysis (Mar – Apr 2024)

Short Description: Pulled data from the Ergast API to explore F1 metrics. Built & deployed a Quarto site for interactive visualizations using Python, FastF1, and SQLite.

Approach: Used Ergast API for driver, race, lap-time, and pit-stop data. Performed extensive EDA and feature creation, visualizing patterns (lap-time deltas, pit strategy comparisons) with advanced Python libraries.

Outcome: Deployed a Quarto-based webpage that includes dashboards, interactive plots, and data insights. Demonstrated how to transform raw motorsport data into meaningful, visually appealing analytics for fans and decision-makers.

Module 3: Reinforcement Learning Endeavors

Short Description: Integrated short-selling thresholds into a DRL stock trading framework. Achieved +11.4% p.a. with ~10% lower max drawdown, second-authored a paper in Springer LNAI.

Objective & Approach: Extended existing DRL-based automated trading architecture to include short positions for volatile market scenarios. Ran multiple experiments under varied market turbulence levels to identify optimal shorting thresholds.

Key Results: Proposed models outperformed older methods by 11.4% annualized returns and a 10% lower max drawdown. This research was presented at the 26th International Symposium on Methodologies for Intelligent Systems (Cosenza, Italy).

6. Deep Reinforcement Learning Based Intelligent Traffic Control (Jul 2022 – Sep 2023)

Short Description: Used DQN, DDPG to optimize traffic signal timing, reducing average waiting times by up to 23%. First-author paper at IEEE TENSYMP 2023.

Approach: Simulated an intersection grid environment, comparing DRL agents (DQN, DDPG) vs. fixed-signal baselines. Achieved a 67% decrease in average wait time over legacy systems, with best results from DQN + environment-specific tuning.

Publication: First-author publication in 2023 IEEE Region 10 Symposium (TENSYMP), Canberra, Australia.

Impact: Real potential for city-scale traffic optimization, reducing congestion, fuel consumption, and pollution. Showcased synergy between RL and transportation engineering.

Module 5: End-to-End Data Projects & Publications

9. Research Paper Recommendation System

Short Description: NLP-based paper recommendation system using TF-IDF, all-MiniLM-L6-v2, and allenai-specter embeddings. Deployed a RAG-based approach with flan-t5 for explainability.

Methodology: Pulled ~100,000 ArXiv papers, preprocessed abstracts (lemmatization, stopword removal), then stored embeddings in LanceDB for scalable retrieval. Fine-tuned for better context alignment, especially for specialized domains (Reinforcement Learning, traffic control).

Explainability: A Retrieval-Augmented Generation (RAG) system with google/flan-t5-large provided natural-language explanations for recommended papers, enhancing user trust and transparency.

Challenges & Outcomes: Balanced resource constraints and scoring optimization. allenai-specter showed strong performance for domain-specific tasks.

10. Commodity Trading using Alternative Data Sources (Weather)

Short Description: Devised coffee trading strategies by integrating weather anomalies with technical indicators (MACD, RSI, Bollinger Bands). Improved robustness and adaptability in volatile markets.

Objective & Approach: Focus on coffee commodity trading from Jan 2020 – Nov 2024. Aggregated hourly weather into daily summaries, capturing frost, drought, rainfall anomalies, and seasonal trends. Combined with standard technical indicators for trend-following + mean-reversion.

Performance Evaluation: Examined Sharpe, Sortino, Calmar ratios, and drawdowns. Strategies using weather data showed more stable returns vs. purely technical approaches, highlighting synergy of exogenous data in mitigating risk.
